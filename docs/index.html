HEO_UPGRADE

비개발자가 Claude Code로 소프트웨어를 만들기 위한 스킬 프레임워크. 막연한 아이디어부터 배포까지, 엔지니어링 전문 지식 없이도 작동하는 품질 게이트를 제공합니다.

v2 20/20 기능 완성 17개 스킬 4단계 Phase

01왜 만들었나

Claude Code는 코드를 대신 써주지만, "돌아가는 코드"와 "믿을 수 있는 코드"는 다릅니다. HEO_UPGRADE는 그 차이를 메웁니다:

  • 다층 검증 — 커밋 전 보안 스캔 + 셀프 리뷰 + 크로스 모델 리뷰
  • 세션 연속성 — 세션 끝에 핸드오프, 시작할 때 자동 복원. 컨텍스트 유실 없음
  • 학습 루프 — 실패와 성공 모두에서 패턴을 추출해 프로젝트를 넘어 축적
  • 투명한 결정 — 모든 단계가 이름 있는 산출물을 남겨, 사용자가 개입하거나 되돌릴 수 있음

05설치 방법

이 저장소를 클론한 뒤, 설치 스크립트를 실행하면 스킬과 규칙이 ~/.claude/로 복사됩니다.

git clone https://github.com/h0130jk2-heo/HEO_UPGRADE.git
cd HEO_UPGRADE

 .\install.ps1        # 처음 설치
.\install.ps1 -Force  # 덮어쓰기
git clone https://github.com/h0130jk2-heo/HEO_UPGRADE.git
cd HEO_UPGRADE

 bash install.sh        # 처음 설치
bash install.sh --force  # 덮어쓰기

기존 설치 업데이트 (v1 → v2)

그냥 설치하면 이미 있는 파일은 건너뛰어서, 새 /reverse-engineer만 추가되고 나머지 스킬은 v1로 남습니다. 바뀐 스킬까지 갱신하려면 최신을 받은 뒤 -Force / --force로 재설치하세요.

# Windows
git pull
.\install.ps1 -Force

# macOS / Linux
git pull
bash install.sh --force

그다음 Claude Code를 재시작하면 새 스킬이 로드됩니다.

학습 데이터는 안전합니다. -Force는 스킬과 정적 규칙(confidence-tags.md, cost-awareness.md)만 새 버전으로 덮어쓰고, instincts.md와 lessons-learned.md는 절대 덮어쓰지 않습니다 — 설치기가 사용자 데이터로 취급해 없을 때만 생성합니다. 업데이트 시 출력에 KEEP (user data preserved)로 표시됩니다. 이 두 파일은 ~/.claude/rules/에 있고 저장소에는 포함되지 않으니, 백업·동기화가 필요하면 직접 복사하세요.

06설계 원칙

1엔지니어링은 독단적으로, 결정은 투명하게

기술적 세부 사항은 프레임워크가 결정하되, 근거를 보여줍니다. 사용자는 방향을 확인하고, 프레임워크가 실행합니다.

2신뢰가 아닌 증거

모든 passes:true는 검증된 것이지, 자기 보고가 아닙니다. 다층 리뷰가 사용자가 잡지 못하는 것을 잡아냅니다.

3실패와 성공 모두에서 배우기

lessons-learned는 실수를, instincts는 잘 된 것을 기록합니다. 둘 다 다음 기능 계획에 반영됩니다.

4단계 게이트, 강제 의식 아님

배포하지 않는 프로젝트는 SHIP 단계를 건너뜁니다. 아이디어가 명확하면 brainstorm을 스킵합니다. 불필요한 부담 없음.

5세션 안전

기능 중간에 작업을 멈춰도 컨텍스트 유실 없이 재개할 수 있습니다. /handoff가 캡처하고, /resume-heo가 복원합니다.

07변경 이력 — v2

v2 (2026-07-06) — "브라운필드 & 엄격성 강화". 신규 스킬 1개와 강화 스킬 4개로 기존 코드베이스 인수, 앱레벨 보안 심화, 요구사항 깊이, 프로바이더 중립 NFR 패턴을 추가했습니다. 배포는 특정 클라우드에 종속되지 않고 프로바이더 중립을 유지하며, HEO 규약(이중언어 산출물, 신뢰도 태그, 경량 opt-in 게이트)을 그대로 지킵니다.
